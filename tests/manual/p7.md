# P7 â€” Log-based diagnostics: manual verification

For environments where vim can't run `make` itself (locked-down
work laptop, build runs on a server, etc.). User points
`g:gccide_log_path` at a stderr log file produced by the build
elsewhere. `:GccideLogRefresh` reads it; `:GccideLogPoll <ms>`
adds an optional timer.

Prerequisites:
- `vim80` (vim 8.0) on PATH.

Steps 1â€“5 are scripted; step 6 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
```

No fixture changes needed. We synthesise log content per step.

## 1. Surfaces reachable

```sh
REPO=$PWD
vim80 -u NONE -i NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'runtime! autoload/gccide/log.vim' \
  -c 'runtime! autoload/gccide/diag.vim' \
  -c 'echo exists("*gccide#log#refresh")' \
  -c 'echo exists("*gccide#log#enable_poll")' \
  -c 'echo exists("*gccide#diag#publish")' \
  -c 'echo exists("*gccide#diag#parse_lines")' \
  -c 'command GccideLogRefresh' \
  -c 'command GccideLogPoll' \
  -c 'qall!' 2>&1
```

**Expect:** four `1` lines, then two `Name â€¦ Args â€¦ Definition â€¦`
rows for `GccideLogRefresh` (0 args) and `GccideLogPoll`
(0 or 1 args).

## 2. Refresh on a synthetic log: signs + quickfix populate

```sh
REPO=$PWD
LOG="$TMPDIR/gccide_p7_build.log"
cat > "$LOG" <<EOF
$REPO/tests/fixtures/proj/src/main.c:5:5: error: implicit declaration of function 'proj_greet'
$REPO/tests/fixtures/proj/src/util.c:8:1: warning: unused parameter 'who'
EOF
cat > "$TMPDIR/gccide_p7_refresh.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_log_path     = '$LOG'
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
edit $REPO/tests/fixtures/proj/src/main.c
GccideLogRefresh
let g:qf = getqflist()
call writefile([
\ 'qf_len=' . len(g:qf),
\ 'qf0=' . string(g:qf[0]),
\ 'qf1=' . string(g:qf[1]),
\ 'signs=' . string(sign_getplaced(bufnr('%'))),
\ ], '$TMPDIR/gccide_p7_refresh.out')
qall!
VIM
rm -f "$TMPDIR/gccide_p7_refresh.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p7_refresh.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p7_refresh.out"
```

**Expect:**
- `qf_len=2`.
- `qf0` contains `bufnr`/`filename` for `main.c`, `lnum: 5`,
  `col: 5`, `type: 'E'`, text containing `error: implicit
  declaration of function 'proj_greet'`.
- `qf1` contains `util.c`, `lnum: 8`, `col: 1`, `type: 'W'`, text
  containing `warning: unused parameter 'who'`.
- `signs=â€¦` includes a `GccideError` sign on `main.c` line 5 (the
  warning on util.c lands in a different buffer, not the loaded
  one â€” vim's `sign_getplaced(bufnr('%'))` only sees signs on the
  active buffer; that is intended).

## 3. Refresh after the log changes: only new diagnostics show

```sh

LOG="$TMPDIR/gccide_p7_build.log"

cat > "$LOG" <<EOF

cat > "$TMPDIR/gccide_p7_replace.vim" <<VIM

GccideLogRefresh
let g:qf = getqflist()
call writefile([
\ 'qf_len=' . len(g:qf),
\ 'qf0=' . string(g:qf[0]),
\ ], '$TMPDIR/gccide_p7_replace.out')
qall!
VIM
rm -f "$TMPDIR/gccide_p7_replace.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p7_replace.vim" </dev/null >/dev/null 2>&1
```

**Expect:** `qf_len=1`, qf0 text contains `expected ';' before '}'`,

## 5. Polling: enable, see mtime tracked, disable

```sh
REPO=$PWD
LOG="$TMPDIR/gccide_p7_build.log"
cat > "$LOG" <<EOF
$REPO/tests/fixtures/proj/src/main.c:5:5: error: poll test
EOF
cat > "$TMPDIR/gccide_p7_poll.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_log_path     = '$LOG'
let g:gccide_project_root = '$REPO/tests/fixtures/proj'

call writefile(['before=' . string(gccide#log#_state())], '$TMPDIR/gccide_p7_poll.out')
call gccide#log#enable_poll(200)
call writefile(['after_enable=' . string(gccide#log#_state())], '$TMPDIR/gccide_p7_poll.out', 'a')
" Sleep long enough for at least one poll tick.
sleep 400m
call writefile(['after_tick=' . string(gccide#log#_state())], '$TMPDIR/gccide_p7_poll.out', 'a')
call gccide#log#disable_poll()
call writefile(['after_disable=' . string(gccide#log#_state())], '$TMPDIR/gccide_p7_poll.out', 'a')
qall!
VIM
rm -f "$TMPDIR/gccide_p7_poll.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p7_poll.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p7_poll.out"
```

**Expect:**
- `before`: `polling: 0`, `poll_ms: 0`, `last_mtime: -1`.
- `after_enable`: `polling: 1`, `poll_ms: 200`. `last_mtime` may
  still be -1 (no poll has fired yet).
- `after_tick`: `polling: 1`, `poll_ms: 200`, `last_mtime`
  reflects the log's actual mtime (a positive integer, the unix
  epoch).
- `after_disable`: `polling: 0`, `poll_ms: 0`.

## 6. Interactive: edit, save log externally, refresh

```sh
REPO=$PWD
LOG="$TMPDIR/gccide_p7_build.log"
cat > "$LOG" <<EOF
$REPO/tests/fixtures/proj/src/main.c:5:5: error: interactive demo error
$REPO/tests/fixtures/proj/src/main.c:8:5: warning: interactive demo warning
EOF
vim80 -u NONE -i NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_log_path     = '$LOG'" \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c"
```

Inside vim:

1. `:GccideLogRefresh` â€” sign columns on lines 5 and 8 (`E>` red
   on line 5, `W>` orange/yellow on line 8). `:copen` shows two
   quickfix entries.
2. Open a second terminal, rewrite the log:
   ```sh
   echo "$REPO/tests/fixtures/proj/src/util.c:4:1: error: another file"  > "$LOG"
   ```
3. Back in vim, `:GccideLogRefresh`. The signs on `main.c` should
   clear (we're not viewing util.c, so no new signs visible).
   `:copen` now shows the single util.c entry.
4. Want polling? `:GccideLogPoll 1000` â€” the plugin re-reads the
   log every second now. Edit the log externally again; within a
   second the quickfix updates without you typing.
5. `:GccideLogPoll 0` stops the timer.
6. `:qa!`.

## Known limitations (intentional for P7)

- **Stale signs are not "merged" across refreshes.** Every
  `:GccideLogRefresh` clears all previously placed signs and
  replaces them with whatever the current log says. That matches
  `make`'s build semantics â€” a clean build either has the error
  or doesn't.
- **Relative paths in the log resolve against `g:gccide_project_root`.**
  If your build emits relative paths and you haven't set the
  project root, signs land on the relative filename (probably
  not what you want). Absolute paths in the log always work.
- **The parser only understands gcc 4â€“8 style stderr.** Clang's
  format is similar; ICC, MSVC, custom build tools may need a
  preprocessor (your `make 2> >(sed | tee log.txt)` pipeline).
- **`:GccideLogPoll` lives on a `timer_start` repeat.** If you
  `:qa` and reopen vim, you have to set it up again. Put
  `let g:gccide_log_poll_ms = 2000` in your vimrc to auto-start.
- **No `make` is run by the plugin in this mode.** This is the
  whole point â€” but it also means you don't get the "build
  succeeded" green-light feedback, only "here are the latest
  errors". Look at the log file's mtime or your build script's
  exit code for success/fail signaling.

## If any step fails

Capture the exact output and append a dated entry to
`docs/JOURNAL.md`. Common gotchas:
- `g:gccide_log_path` unset or pointing at a non-existent file.
- Log entries use a path that doesn't match any loaded buffer â€”
  signs only render in loaded buffers; the quickfix list always
  fills.
- Polling started but the file is on a network mount with weird
  mtime semantics â€” fall back to manual `:GccideLogRefresh`.
