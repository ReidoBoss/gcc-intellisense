# P4 — Identifier autocomplete: manual verification

The plugin installs `omnifunc=gccide#complete#omnifunc` on
`FileType c,cpp` buffers. Triggering omnicomplete (i_CTRL-X i_CTRL-O)
returns identifier-prefix-filtered candidates drawn from the index
built by P3. The candidate source is pluggable
(`gccide#complete#register_source(Funcref)`) so a future semantic
backend can slot in without touching the omnifunc surface.

Prerequisites:
- `vim80` (vim 8.0) on PATH.
- `find` and `sh` — standard.

Steps 1–6 are scripted; step 7 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
rm -rf tests/fixtures/proj/.gccide
git checkout -- tests/fixtures/proj/src/main.c
```

The fixture has 5 indexable symbols: `PROJ_H`, `PROJ_MAGIC` (defines
in `inc/proj.h`), `main` (function in `src/main.c`), `proj_add` and
`proj_greet` (functions in `src/util.c`). Prefix `proj_` matches two
of them; prefix `PROJ_` matches the other two.

Scripted steps call the omnifunc directly
(`gccide#complete#omnifunc(0, '<prefix>')`) — interactive `<C-x><C-o>`
completion is hard to drive from `vim80 -S`. Step 7 covers the
interactive path.

## 1. Plugin loads, omnifunc is reachable

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'echo exists("*gccide#complete#omnifunc")' \
  -c 'echo exists("*gccide#complete#register_source")' \
  -c 'echo exists("*gccide#index#candidates")' \
  -c 'qa!' 2>&1
```

**Expect:** three `1` lines.

## 2. Empty index: no candidates, helpful nudge

Without an on-disk `.gccide/index` and without calling `:GccideIndex`,
the omnifunc returns `[]` and echoes a one-liner pointing at
`:GccideIndex`.

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
cat > "/tmp/gccide_p4_empty.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
let g:res = gccide#complete#omnifunc(0, 'proj_')
call writefile(['len=' . len(g:res)], '/tmp/gccide_p4_empty.out')
qa!
VIM
rm -f "/tmp/gccide_p4_empty.out"
vim80 -u NONE -N -S "/tmp/gccide_p4_empty.vim" </dev/null 2>&1 | grep -E 'gccide:|^len' || true
cat "/tmp/gccide_p4_empty.out"
```

**Expect:**
- A `gccide: index empty (run :GccideIndex)` line in the captured
  output (the warning fires when the result list is empty and no
  index is loaded).
- `len=0`.

## 3. Build index, then prefix-match `proj_`

```sh
REPO=$PWD
cat > "/tmp/gccide_p4_proj.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideIndex
call gccide#index#_wait_done(10000)
let g:res = gccide#complete#omnifunc(0, 'proj_')
call writefile(map(copy(g:res), {_, c -> c.word . '|' . c.kind . '|' . c.menu}),
      \ '/tmp/gccide_p4_proj.out')
qa!
VIM
rm -f "/tmp/gccide_p4_proj.out"
vim80 -u NONE -N -S "/tmp/gccide_p4_proj.vim" </dev/null >/dev/null 2>&1
cat "/tmp/gccide_p4_proj.out"
```

**Expect:** exactly two lines (sorted):

```
proj_add|f|util.c
proj_greet|f|util.c
```

`f` is the `kind` field (function); `menu` shows the source file's
tail. If the same symbol appeared in more than one file, the menu
would end with ` +N` where N is the extra-hit count.

```sh
REPO=$PWD
cat > "/tmp/gccide_p4_upper.vim" <<VIM

      \ '/tmp/gccide_p4_upper.out')

VIM
rm -f "/tmp/gccide_p4_upper.out"
vim80 -u NONE -N -S "/tmp/gccide_p4_upper.vim" </dev/null >/dev/null 2>&1

```

**Expect:** exactly two lines:


`d` is the `kind` field (define). The index is auto-loaded from disk
on this run — no `:GccideIndex` needed because step 3 persisted
`.gccide/index`.

## 6. findstart returns the prefix column correctly

```sh
REPO=$PWD
cat > "/tmp/gccide_p4_start.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
edit $REPO/tests/fixtures/proj/src/main.c
call cursor(5, 10)
" Cursor sits inside 'proj_greet' on line 5. Column 10 lands somewhere
" inside the identifier — findstart must return the column where the
" identifier begins (1-based start position - 1, i.e. the 0-based col
" vim hands to the omnifunc).
let g:start = gccide#complete#omnifunc(1, '')
call writefile(['start=' . g:start, 'line=' . getline(5)],
      \ '/tmp/gccide_p4_start.out')
qa!
VIM
rm -f "/tmp/gccide_p4_start.out"
vim80 -u NONE -N -S "/tmp/gccide_p4_start.vim" </dev/null >/dev/null 2>&1
cat "/tmp/gccide_p4_start.out"
```

**Expect:**
```
start=4
line=    proj_greet("world");
```

Line 5 of `main.c` is `    proj_greet("world");`. The identifier
`proj_greet` starts at the 5th character (0-based col 4). `findstart`
must return `4`.

## 7. Interactive: omnicomplete in a real buffer

Launch vim on `main.c` (the `-u NONE -i NONE` flags skip your vimrc
and viminfo so this is reproducible; the `-c 'filetype plugin on'`
re-enables filetype detection that `-u NONE` disabled, which is what
fires our `FileType c` autocmd):

```sh
REPO=$PWD
vim80 -u NONE -i NONE -N \
  -c 'filetype plugin on' \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "let g:gccide_source_root  = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c"
```

Then, step by step inside vim:

1. **Build the index.** Type `:GccideIndex` and press `Enter`. Wait
   for `gccide: index built (5 symbols, 3 files)` to appear at the
   bottom.

2. **Verify the omnifunc is set.** Type `:setlocal omnifunc?` and
   press `Enter`. Expect:
   ```
   omnifunc=gccide#complete#omnifunc
   ```
   If it prints `omnifunc=` (empty), the `FileType c` autocmd
   didn't fire. Run `:setlocal omnifunc=gccide#complete#omnifunc`
   manually to continue, and capture the failure in
   `docs/JOURNAL.md` afterward.

3. **Open a fresh insert line.** Move the cursor to the
   `    return 0;` line. Press `o` (lowercase letter o) — vim
   opens a new line below and drops you into insert mode.

4. **Trigger completion on a lowercase prefix.** Type the five
   characters `p` `r` `o` `j` `_`, then press `Ctrl-X` followed by
   `Ctrl-O`. A popup appears showing exactly two entries:
   ```
   proj_add    f util.c
   proj_greet  f util.c
   ```
   The `f` is the kind indicator (function); `util.c` is the
   source file.

5. **Navigate the popup.** `Ctrl-N` highlights the next entry;
   `Ctrl-P` the previous. As you move, the buffer previews the
   highlighted candidate.

6. **Accept or dismiss.** `Ctrl-Y` (or `Enter`) inserts the
   highlighted word; `Ctrl-E` dismisses the popup leaving only
   the prefix you typed.

7. **Trigger completion on an uppercase prefix.** Press `Esc` to
   leave insert mode, then `o` to open another new line in insert
   mode. Type `P` `R` `O` `J` `_`, then `Ctrl-X Ctrl-O`. Popup:
   ```
   PROJ_H      d proj.h
   PROJ_MAGIC  d proj.h
   ```
   The `d` is the kind indicator (define).

8. **Quit without saving.** `Esc`, then `:q!`, then `Enter`.

**Common failure modes:**
- Nothing happens when you press `Ctrl-X Ctrl-O` → you're probably
  in normal mode. Insert mode only.
- Popup shows but the entries are wrong → re-run `:GccideIndex` and
  retry; the disk index may be stale from an older fixture.
- `Ctrl-X Ctrl-O` produces `pattern not found` → the prefix doesn't
  match anything in the index. Sanity-check with `:GccideFind
  proj_add` first.

## Known limitations (intentional for P4)

- **No implicit `:GccideIndex` on first complete.** The build is
  async; the first omnicomplete would return `[]` regardless. We
  echo a nudge instead. Explicit is better than surprising.
- **No per-buffer re-extract on save.** A freshly-typed identifier
  is not visible to completion until the next full `:GccideIndex`.
  Incremental re-index is queued for P6.
- **Header guards (`PROJ_H`) show up as `d` candidates.** They are
  real `#define`s — filtering them needs a heuristic (e.g. uppercase
  + matching `#ifndef` on the line above). Polish for a later phase.
- **Prefix match is case-sensitive.** This mirrors vim's built-in
  omnicomplete behavior. If we ever want `ignorecase`-aware matching,
  it goes in `gccide#index#candidates()`.

## If any step fails

Capture the exact output and append to `docs/JOURNAL.md` under a new
dated entry. Common gotchas:
- `g:gccide_project_root` (or `g:gccide_source_root`) unset.
- `.gccide/index` missing — run `:GccideIndex` and wait for the
  `index built …` echo before triggering completion.
- `:setlocal omnifunc?` prints `omnifunc=` (empty) — the `FileType
  c,cpp` autocmd didn't fire. `vim80 -u NONE` skips filetype
  detection; step 7's launch command must include
  `-c 'filetype plugin on'` before `-c "edit …main.c"`. A real
  user's vimrc has filetype detection on, so this is a test-rig
  issue, not a plugin issue.
