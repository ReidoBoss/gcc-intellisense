# P5 — Go-to-definition: manual verification

`gccide#goto#def()` looks up `expand('<cword>')` in the P3 identifier
index.

- Single hit, target in **another** file → opens it in a new tab
  (default `tabedit` — override via `g:gccide_split_cmd`, e.g.
  `'split'` for a horizontal split or `'vsplit'` for vertical) and
  jumps the cursor to `(lnum, col)`.
- Single hit, target in the **current** file → jumps in place,
  same window, no tab opened. The prior cursor position is pushed
  onto the jumplist via the `<lnum>G` motion, so `<C-o>` walks
  back to where you triggered the goto from.
- Multiple hits → populates the quickfix with every hit and `:copen`s,
  then jumps to the first.
- Zero hits → echoes `gccide: no definition for <word>`.
- Cursor already on the definition line → echoes
  `gccide: already at definition of <word>` and does **not** open a
  split.

User surfaces:
- `<Plug>(gccide-goto-def)` — the canonical mapping target.
- `gd` (normal mode) — installed automatically if no user mapping
  exists and `g:gccide_auto` is on. This **shadows vim's built-in
  `gd`** (goto local declaration). Vim's built-in is rarely useful
  inside multi-file C projects — our `gd` jumps to the global
  definition across the index. If you want the original back,
  either `unmap gd` or map `<Plug>(gccide-goto-def)` to a
  different key in your vimrc before this plugin loads.
- `:GccideGoto` — command form.

Prerequisites:
- `vim80` (vim 8.0) on PATH.

Steps 1–6 are scripted; step 7 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
rm -rf tests/fixtures/proj/.gccide
git checkout -- tests/fixtures/proj/src/main.c
```

Fixture symbols (from P3): `PROJ_H`, `PROJ_MAGIC` (defines in
`inc/proj.h`), `main` (function in `src/main.c`), `proj_add` and
`proj_greet` (functions in `src/util.c`). The header has
*declarations* only — go-to-def will land you in `util.c` where the
definitions live.

## 1. Plugin loads, surfaces are reachable

```sh
REPO=$PWD
vim80 -u NONE -i NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'runtime! autoload/gccide/goto.vim' \
  -c 'runtime! autoload/gccide/index.vim' \
  -c 'echo exists("*gccide#goto#def")' \
  -c 'echo exists("*gccide#index#lookup")' \
  -c 'command GccideGoto' \
  -c 'qall' 2>&1
```

**Expect:** two `1` lines, then a `Name … Args … Definition …` row
for `GccideGoto`.

## 2. Single hit: `proj_greet` called from main.c lands on its
   definition in util.c

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
cat > "$TMPDIR/gccide_p5_single.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideIndex
call gccide#index#_wait_done(10000)
edit $REPO/tests/fixtures/proj/src/main.c
" Line 5: '    proj_greet("world");'. Land cursor inside the call.
call cursor(5, 10)
call gccide#goto#def()
call writefile([
\ 'bufname=' . fnamemodify(bufname('%'), ':t'),
\ 'lnum=' . line('.'),
\ 'col=' . col('.'),
\ 'tabcount=' . tabpagenr('$'),
\ 'tabnr=' . tabpagenr(),
\ ], '$TMPDIR/gccide_p5_single.out')
qall
VIM
rm -f "$TMPDIR/gccide_p5_single.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p5_single.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p5_single.out"
```

**Expect:**
```
bufname=util.c
lnum=8
col=1
tabcount=2
tabnr=2
```

`util.c` line 8 is `void proj_greet(const char *who) {`. A new tab
was opened (`tabcount=2`) and the cursor is now in it (`tabnr=2`).

## 3. Same-file hit: jumps in place, no new tab

The fixture has no same-file usage of any symbol, so this step
injects a phantom line into the live buffer via `append()` (the
file on disk is **not** modified — the index keeps pointing at
the original definition). With the cursor on that phantom
`proj_add`, goto should stay in `util.c` without spawning a tab.

```sh
REPO=$PWD
cat > "$TMPDIR/gccide_p5_samefile.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
edit $REPO/tests/fixtures/proj/src/util.c
call append(line('\$'), ['proj_add'])
let g:phantom = line('\$')
call cursor(g:phantom, 1)
call gccide#goto#def()
call writefile([
\ 'tabcount=' . tabpagenr('\$'),
\ 'bufname=' . fnamemodify(bufname('%'), ':t'),
\ 'lnum=' . line('.'),
\ ], '$TMPDIR/gccide_p5_samefile.out')
qall!
VIM
rm -f "$TMPDIR/gccide_p5_samefile.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p5_samefile.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p5_samefile.out"
```

**Expect:**
```
tabcount=1
bufname=util.c
lnum=4
```

No new tab. `util.c` line 4 is `int proj_add(int a, int b) {`,
so the in-place cursor move is correct. After this jump, pressing
`<C-o>` (interactively, not in batch) would return to the phantom
`proj_add` line — that's the jumplist working.

call cursor(6, 6)

call writefile([
\ 'tabcount=' . tabpagenr('$'),

**Expect:**
- A `gccide: no definition for printf` line in the captured output.

## 7. Interactive: real `gd` flow

```sh
REPO=$PWD
vim80 -u NONE -i NONE -N \
  -c 'filetype plugin on' \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "let g:gccide_source_root  = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c"
```

Then in vim:

1. **Build the index.** `:GccideIndex` and wait for
   `gccide: index built (5 symbols, 3 files)`.

2. **Position cursor on a callee.** Move to line 5 (`proj_greet("world");`).
   Use `5G` to jump there, then `w` once to land on `proj_greet`.

3. **Trigger go-to-def.** Press `gd` (normal mode, two keystrokes:
   lowercase `g` then `d`). Vim opens a new tab with `util.c`,
   cursor sits on line 8 at `void proj_greet(...) {`. The tab line
   at the top shows both `main.c` and `util.c`; you're on the
   `util.c` tab. `:tabnext` / `:tabprevious` (or `gt` / `gT`) walk
   between them.

4. **Close the tab and try another.** `:tabclose` to dismiss the
   new tab — you're back on `main.c`. Move to line 6, on
   `PROJ_MAGIC` (use `6G` then `f P`). Press `gd`. A new tab
   opens on `inc/proj.h` at line 4 with
   `#define PROJ_MAGIC 0xC0FFEE`.

5. **Try the no-hit path.** `:tabclose`. Move to `printf` on line 6
   (`6G` then `^` to first non-blank, then `f p`). Press `gd`.
   No new tab opens; the echo line reads
   `gccide: no definition for printf`.

6. **Try the already-at-definition path.** Close any extra tabs
   so only `main.c` is open. Move to line 4 (`int main(void) {`),
   land on `main` (`4G^w` or similar). Press `gd`. No new tab
   opens; echo reads `gccide: already at definition of main`.

7. **Try the same-file path.** `:tabclose` any leftover tabs, then
   `:tabnew` and `:edit tests/fixtures/proj/src/util.c` (or just
   open it however). The fixture has no real same-file callers, so
   simulate one: in insert mode at end-of-file, type `proj_add`
   on a new line, then `Esc` (do NOT `:w`). Press `gd` on that
   `proj_add`. Vim **stays in the same window** (no new tab) and
   the cursor moves to line 4 (`int proj_add(...)`). Press
   `<C-o>` — cursor jumps back to your phantom `proj_add` line.
   Press `u` to undo the phantom line. `<C-o>` confirms the
   jumplist is working as expected.

8. **Quit.** `:qa!`.

## Coverage gap

The bundled fixture has no symbol with two definitions, so the
multi-hit branch of `gccide#goto#def()` (populate quickfix +
`:copen` + `:cfirst`) is **not** exercised by this checklist. The
code path mirrors `:GccideFind`'s quickfix wiring (see
`autoload/gccide/goto.vim`). Verify against a real codebase that has
e.g. a function defined in two `#ifdef` arms.

## Known limitations (intentional for P5)

- **Header declaration vs definition.** A function declared in a
  `.h` file with no body never reaches the index, so go-to-def
  always lands on the body in the `.c`. That is the right behavior
  for now; if you ever want "show me both", we'd add a separate
  index kind for declarations in P6+ and a `:GccideGotoDecl` cousin.
- **Cursor word is whatever `<cword>` returns.** If the cursor is
  on whitespace or a punctuation char, `expand('<cword>')` may
  return an empty string or the wrong identifier. We echo
  `gccide: no word under cursor` for the empty case.
- **Default open command is `tabedit`** (new tab). Set
  `let g:gccide_split_cmd = 'split'` (horizontal) or `'vsplit'`
  (vertical) in your vimrc to change it.
- **The default `gd` mapping is skipped** if either
  `<Plug>(gccide-goto-def)` is already mapped somewhere or `gd`
  is already user-mapped. Vim's built-in `gd` (goto local
  declaration) is **not** a user mapping — `maparg('gd', 'n')`
  returns empty for the built-in — so the plugin will install on
  top of it. Use `<Plug>(gccide-goto-def)` directly from your
  vimrc to pick your own key.
- **Same as P4**: no implicit `:GccideIndex` on first call. If the
  in-memory + on-disk index are both empty, every lookup reports
  "no definition". Run `:GccideIndex` once per session (it persists
  to disk so a re-launch picks it up).

## If any step fails

Capture the exact output (or `:messages` for interactive) and
append a dated entry to `docs/JOURNAL.md`. Common gotchas:
- `g:gccide_project_root` (or `g:gccide_source_root`) unset.
- `.gccide/index` missing or stale — run `:GccideIndex`.
- Cursor not actually on the identifier (`<cword>` returns
  something unexpected). `:echo expand('<cword>')` to verify.
