# P2 — Diagnostics (via make): manual verification

The diagnostic engine runs your project's `Makefile` (not
`gcc -fsyntax-only` per buffer). On `:w` we kick off
`g:gccide_make_cmd` (default `'make'`) in `g:gccide_project_root` and
parse gcc's stderr from there. Signs land in any loaded buffer that
matches a diagnostic's absolute path; everything goes into the
quickfix list.

Prerequisites:
- `vim80` (vim 8.0) on PATH.
- A real `gcc` 8.5.0 (or compatible) reachable by your Makefile.
  The plugin itself never invokes gcc directly — make does.

Steps 1–5 are scripted; step 6 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
rm -rf tests/fixtures/proj/.gccide
git checkout -- tests/fixtures/proj/src/main.c   # baseline-clean fixture
```

`gccide#diag#_wait_done(timeout_ms)` is a test-only helper. In scripted
(`vim80 -S`) mode, plain `:sleep` doesn't reap exited make children —
`_wait_done` polls `job_status()` until the `exit_cb` has fired.
Interactive vim (step 6) doesn't need it.

## 1. Plugin still loads, :GccideStatus and :GccideFlags work

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c" \
  -c 'GccideStatus | GccideFlags | sleep 1500m | qa!'
```

**Expect:** `alive` printed, then the P1 "extracting…" and "flags
ready…" messages. No `E*` errors.

## 2. :GccideDiag without `g:gccide_project_root` warns and bails

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "edit $REPO/tests/fixtures/proj/src/main.c | GccideDiag | qa!" 2>&1 | grep gccide
```

**Expect:** a line `gccide: g:gccide_project_root not set ...`.

## 3. Clean fixture → make succeeds, zero diagnostics

```sh
REPO=$PWD
cat > /tmp/gccide_p2_clean.vim <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
edit $REPO/tests/fixtures/proj/src/main.c
GccideDiag
call gccide#diag#_wait_done(10000)
let g:s = ''
redir => g:s
silent execute 'sign place buffer=' . bufnr('%')
redir END
call writefile(['---signs---'] + split(g:s, "\n") + ['---qf---', string(getqflist())], '/tmp/gccide_p2_clean.out')
qa!
VIM
rm -f /tmp/gccide_p2_clean.out
vim80 -u NONE -N -S /tmp/gccide_p2_clean.vim </dev/null >/dev/null 2>&1
cat /tmp/gccide_p2_clean.out
```

**Expect:**
```
---signs---
--- Signs ---
---qf---
[]
```

(The fixture's `make` will build `proj` — a small binary may appear
under `tests/fixtures/proj/`; that's fine, it's git-ignored.)

## 4. Injected error → make fails, signs + quickfix items

Inject a real syntax error into `main.c`, run the diagnostic, then
restore the fixture from git.

```sh
REPO=$PWD
cat > "$REPO/tests/fixtures/proj/src/main.c" <<'C'
#include <stdio.h>
#include "proj.h"

int main(void) {
    proj_greet("world");
    printf("magic = 0x%x\n", PROJ_MAGIC);
    printf("sum   = %d\n", proj_add(2, 3));
    bad_token_xyz();
    return 0;
}
C
cat > /tmp/gccide_p2_err.vim <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
edit $REPO/tests/fixtures/proj/src/main.c
GccideDiag
call gccide#diag#_wait_done(10000)
let g:s = ''
redir => g:s
silent execute 'sign place buffer=' . bufnr('%')
redir END
call writefile(['---signs---'] + split(g:s, "\n") + ['---qf-len---', string(len(getqflist())), '---qf---', string(getqflist())], '/tmp/gccide_p2_err.out')
qa!
VIM
rm -f /tmp/gccide_p2_err.out
vim80 -u NONE -N -S /tmp/gccide_p2_err.vim </dev/null >/dev/null 2>&1
cat /tmp/gccide_p2_err.out
git checkout -- "$REPO/tests/fixtures/proj/src/main.c"
```

**Expect:**
- At least one `name=GccideError` sign on `main.c` (likely `line=8` —
  the `bad_token_xyz();` call).
- `qf-len` ≥ 1. The first qf item has `'type': 'E'`, lnum 8, and text
  starting with `error: ` mentioning `bad_token_xyz`.

(Different gcc versions phrase the message differently — what matters
is the severity is `error`, the file is `main.c`, and the line is 8.)

## 5. :GccideDiagClear removes signs and clears quickfix

```sh
REPO=$PWD
cat > "$REPO/tests/fixtures/proj/src/main.c" <<'C'
#include <stdio.h>
#include "proj.h"

int main(void) {
    bad_token_for_clear_test();
    return 0;
}
C
cat > /tmp/gccide_p2_clear.vim <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
edit $REPO/tests/fixtures/proj/src/main.c
GccideDiag
call gccide#diag#_wait_done(10000)
let g:before = ''
redir => g:before
silent execute 'sign place buffer=' . bufnr('%')
redir END
GccideDiagClear
let g:after = ''
redir => g:after
silent execute 'sign place buffer=' . bufnr('%')
redir END
call writefile(
      \ ['---before---'] + split(g:before, "\n") +
      \ ['---after---']  + split(g:after,  "\n") +
      \ ['---qf-after---', string(getqflist())],
      \ '/tmp/gccide_p2_clear.out')
qa!
VIM
rm -f /tmp/gccide_p2_clear.out
vim80 -u NONE -N -S /tmp/gccide_p2_clear.vim </dev/null >/dev/null 2>&1
cat /tmp/gccide_p2_clear.out
git checkout -- "$REPO/tests/fixtures/proj/src/main.c"
```

**Expect:**
- `---before---` section has at least one `name=GccideError` line.
- `---after---` section has no `line=` entries (just `--- Signs ---`).
- `---qf-after---` is `[]`.

## 6. Interactive: `:w` triggers make and surfaces errors

Open the fixture's `main.c` in vim normally:

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c"
```

Then:

1. `G o` to open a new line at the bottom of the function.
2. Type `bad_token;` and `Esc`.
3. `:w` to save.

**Expect:** within ~half a second of the save (make + 100 ms pulse),
a `E>` sign appears on the line you just added. `:copen` opens a
quickfix window listing the diagnostic.

4. `:cclose`, then `u` (undo) to remove `bad_token;`, then `:w` again.

**Expect:** sign disappears and `:copen` shows an empty list.

5. `:qa!` to leave.

```sh
# Whatever state main.c ended up in on disk, restore baseline:
git checkout -- "$REPO/tests/fixtures/proj/src/main.c"
```

## If any step fails

Capture the exact output (or the `:messages` buffer if interactive)
and append to `docs/JOURNAL.md` under a new dated entry before
handing off. Common gotchas:
- `g:gccide_project_root` unset or pointing somewhere without a Makefile.
- `g:gccide_make_cmd` defaults to `'make'`; on a real codebase you may
  need `let g:gccide_make_cmd = '/path/to/your/script.sh'` or
  `'make some_target'` instead. The command runs from inside
  `g:gccide_project_root` (we `cd` first).
- Diagnostic absolute paths must match buffer abs paths exactly. If
  signs don't appear on a loaded buffer but the quickfix has the
  diagnostic, the make output probably emitted a different path
  spelling (symlinks, relative paths, etc).
