# P2 — Diagnostics: manual verification

Runs on the Mac dev box against the bundled fixture at
`tests/fixtures/proj/`. Steps 1–7 are scripted; steps 8–9 are
interactive (open vim normally and watch the gutter).

## Setup

```sh
cd /Users/stephensagarino/Projects/gcc-only-ide
command -v vim80   # /Users/stephensagarino/opt/vim80/bin/vim
test -x /Users/stephensagarino/Personal-Binaries/xpack-gcc-8.5.0-1/bin/gcc && echo gcc-ok
rm -rf tests/fixtures/proj/.gccide
```

The plugin needs the real path to gcc 8.5.0 in `g:gccide_gcc`; `gcc85`
is a shell alias and does not survive `job_start()`. All test scripts
below `let g:gccide_gcc = '/Users/.../gcc'` themselves; in your real
vimrc, do the same once.

`gccide#diag#_wait_done(bufnr, timeout_ms)` is a test-only helper.
In scripted (`vim80 -S`) mode, plain `:sleep` doesn't reap exited
gcc children — `_wait_done` polls `job_status()` until the exit_cb has
fired. Interactive vim (steps 8–9) doesn't need this.

## 1. Plugin still loads, :GccideStatus and :GccideFlags still work

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'GccideStatus | edit tests/fixtures/proj/src/main.c | GccideFlags | sleep 1500m | qa!'
```

**Expect:** `alive` printed first, then the P1 "extracting…" and
"flags ready…" messages. No `E*` errors.

## 2. :GccideDiag without `g:gccide_gcc` warns and bails

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit tests/fixtures/proj/src/main.c | GccideDiag | qa!' 2>&1 | grep gccide
```

**Expect:** a line `gccide: g:gccide_gcc is not set (e.g. let g:gccide_gcc = "/path/to/gcc")`.
No `E*` errors, no signs placed.

No diagnostics for a clean file.

## 5. Header inherits TU flags

`inc/proj.h` does not appear in `make -Bnk` output. `for_file()` falls
back to flags from any TU in the project so headers still get checked.

```sh

" Prime the flag cache via main.c, then switch to the header.
edit /Users/stephensagarino/Projects/gcc-only-ide/tests/fixtures/proj/src/main.c
GccideFlags
sleep 1500m
edit /Users/stephensagarino/Projects/gcc-only-ide/tests/fixtures/proj/inc/proj.h
let g:b = bufnr('%')
call append(line('$'), 'broken broken broken;')
GccideDiag
call gccide#diag#_wait_done(g:b, 5000)
let g:s = ''
redir => g:s
silent execute 'sign place buffer=' . g:b
redir END
call writefile(['---signs---'] + split(g:s, "\n") + ['---qf-len---', string(len(getqflist()))], '/tmp/gccide_p2_hdr.out')

vim80 -u NONE -N -S /tmp/gccide_p2_hdr.vim </dev/null >/dev/null 2>&1

```

**Expect:** at least 1 `GccideError` sign and `qf-len` ≥ 1. The
diagnostic comes from gcc parsing `broken broken broken;` in the
header — confirming the header found the project's `-I`/`-D` flags.

```sh
cat > /tmp/gccide_p2_clear.vim <<'VIM'
set rtp+=/Users/stephensagarino/Projects/gcc-only-ide

GccideDiag

let g:before = ''
redir => g:before
silent execute 'sign place buffer=' . g:b
redir END
GccideDiagClear

call writefile(
      \ ['---before---'] + split(g:before, "\n") +
      \ ['---after---']  + split(g:after,  "\n") +
      \ ['---qf-after---', string(getqflist())],
      \ '/tmp/gccide_p2_clear.out')

vim80 -u NONE -N -S /tmp/gccide_p2_clear.vim </dev/null >/dev/null 2>&1
cat /tmp/gccide_p2_clear.out
```

**Expect:**
- `---before---` section has at least one `name=GccideWarning` line.
- `---after---` section has no `line=` entries (just `--- Signs ---`).
- `---qf-after---` is `[]`.

## 7. Severity matrix

Cause gcc to emit an actual `error:` (not just warning). Touch the
fixture's header to introduce a syntax error there, then check signs
on `main.c` (which includes it).

```sh
cat > /tmp/gccide_p2_sev.vim <<'VIM'
set rtp+=/Users/stephensagarino/Projects/gcc-only-ide
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_gcc = '/Users/stephensagarino/Personal-Binaries/xpack-gcc-8.5.0-1/bin/gcc'
edit /Users/stephensagarino/Projects/gcc-only-ide/tests/fixtures/proj/src/main.c
GccideFlags
sleep 1500m
let g:b = bufnr('%')
" Replace `int main(void) {` with a deliberately broken line.
silent! %s/int main(void) {/int main(void) { undeclared_thing()/
GccideDiag
call gccide#diag#_wait_done(g:b, 5000)
let g:s = ''
redir => g:s
silent execute 'sign place buffer=' . g:b
redir END
call writefile(['---signs---'] + split(g:s, "\n") + ['---qf-types---'] + map(getqflist(), 'v:val.type'), '/tmp/gccide_p2_sev.out')
qa!
VIM
rm -f /tmp/gccide_p2_sev.out
vim80 -u NONE -N -S /tmp/gccide_p2_sev.vim </dev/null >/dev/null 2>&1
cat /tmp/gccide_p2_sev.out
```

**Expect:** at least one `GccideError` sign and the `---qf-types---`
section contains at least one line `E`.

## 8. Interactive: live TextChanged debounce (opt-in)

Live diagnostics during typing are **off by default** — `gcc
-fsyntax-only` on a large TU can take seconds, and re-running it on
every TextChanged debounce is wasteful. Set `g:gccide_live = 1` to opt
in. Without it, only `:w` (step 9) triggers a re-check.

The gutter column is vim's "sign column" — it shows `W>`/`E>`/`N>`
markers from this plugin. `:set number` shows absolute line numbers
next to it (unrelated to diagnostics).

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_gcc = '/Users/stephensagarino/Personal-Binaries/xpack-gcc-8.5.0-1/bin/gcc'" \
  -c 'let g:gccide_live = 1' \
  -c 'edit tests/fixtures/proj/src/main.c' \
  -c 'GccideFlags'
```

Wait for `gccide: flags ready ...`. Then:

1. Press `G` to go to the last line, then `o` to open a new line.
2. Type `xyz_not_a_token;` and press `Esc`.
3. Wait ~300 ms.

**Expect:** within ~500 ms a `W>` (or `E>`) sign appears in the gutter
on the line you just added (300 ms debounce + the gcc run itself).
`:copen` opens a quickfix window listing the diagnostic. `:cclose` to
close it.

If the sign takes multiple seconds to appear, `g:gccide_gcc` may be
pointing at the wrong binary, or the patched `out_io='pipe'` change
isn't loaded — re-source `plugin/gccide.vim` and try again.

4. Replace the bad line: `cc` then type `printf("ok\n");` and press `Esc`.

**Expect:** after ~300 ms the sign disappears and `:copen` shows an
empty quickfix list.

`:qa!` to leave (don't save).

## 9. Interactive: BufWritePost re-runs diagnostics on save

Same vim invocation as step 8 (or relaunch). With clean code in the
buffer and no signs, do:

1. `G o` to open a new line.
2. Type `bad_token;` and `Esc`.
3. `:w` to save.

**Expect:** sign appears in the gutter within ~half a second of the
save (the save triggers `BufWritePost` → `gccide#diag#run` immediately,
plus the gcc run itself).

4. `:qa!` to discard the dirty buffer (do **not** try to undo + save —
   trailing-newline normalization can permanently mutate the fixture).
5. Restore the fixture from git, since step 3 wrote `bad_token;` to disk:

   ```sh
   git checkout -- tests/fixtures/proj/src/main.c
   ```

## Tear-down

```sh
rm -rf tests/fixtures/proj/.gccide
rm -f /tmp/gccide_p2_*.vim /tmp/gccide_p2_*.out
```

(The fixture's source files are unchanged; only the auto-extracted

## If any step fails

Capture the exact output (or the `:messages` buffer if interactive)
and append to `docs/JOURNAL.md` under a new dated entry before
handing off. Common gotchas:
- `g:gccide_gcc` set to the shell alias `gcc85` instead of the real
  binary path — the plugin will warn, but a stale vimrc may have it
  wrong.
- `tests/fixtures/proj/.gccide/flags` cached from a previous run with
  an old Makefile — `rm -rf` it before re-running.
