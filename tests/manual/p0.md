# P0 — Skeleton: manual verification

Run these on the **Mac dev box**. The Mac has `vim80` aliased to real
vim 8.0; no work laptop required.

## Setup

```sh
cd /Users/stephensagarino/Projects/gcc-only-ide
command -v vim80   # expect: alias for /Users/stephensagarino/opt/vim80/bin/vim
```

All commands below run from the repo root, and rely on `set rtp+=.` to
make vim see this checkout.

## 1. Plugin loads without error

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'echo "loaded ok"' \
  -c 'qa!'
```

**Expect:** `loaded ok` printed. No `E*` errors. No tracebacks.

## 2. `:GccideStatus` returns `alive`

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'redir! > /tmp/gccide_p0.out' \
  -c 'GccideStatus' \
  -c 'redir END' \
  -c 'qa!'
grep -F alive /tmp/gccide_p0.out && echo OK || echo FAIL
rm -f /tmp/gccide_p0.out
```

**Expect:** final line is `OK`.

## 3. Load guard works (double-source is a no-op)

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'echo g:loaded_gccide' \
  -c 'qa!'
```

**Expect:** prints `1`. No `E174: Command already exists` from the second
`runtime!`.

## 4. Autoload resolves on demand

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'echo gccide#status()' \
  -c 'qa!'
```

**Expect:** prints `alive`. Vim located `autoload/gccide.vim` without
`plugin/gccide.vim` being sourced first.

## If any step fails

Capture the exact output and append it to `docs/JOURNAL.md` under a new
entry before handing off.
