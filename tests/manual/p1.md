# P1 — Project detection + flag extraction: manual verification

Prerequisites: `vim80` and `make` on PATH. Runs against the bundled
fixture at `tests/fixtures/proj/`; nothing external required. Works on
any machine.

## Setup

```sh
cd /path/to/your/gcc-only-ide    # this repo's checkout
command -v vim80 make            # both must exist
rm -rf tests/fixtures/proj/.gccide   # start clean
```

Fixture layout:

```
tests/fixtures/proj/
├── Makefile
├── inc/proj.h
└── src/
    ├── main.c
    └── util.c
```

The Makefile compiles two TUs with `-Iinc -DPROJ_BUILD=1 -DVERSION=\"0.1\"
-std=c99 -Wall -Wextra`. P1 only needs `make -Bnk` (a dry run), so no
binary is actually built and `gcc` itself never runs at this stage.

The extraction is **async** (`job_start` + `make -Bnk`). On this fixture
it should complete in well under a second; on a large firmware tree it
can take several seconds. Wait for the `gccide: flags ready ...` message
before re-running `:GccideFlags`.

## 1. Plugin still loads, `:GccideStatus` still works

```sh
vim80 -u NONE -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'GccideStatus' \
  -c 'qa!'
```

**Expect:** `alive` printed. No `E*` errors.

## 2. `:GccideFlags` discovers the Makefile and extracts flags

```sh
vim80 -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit tests/fixtures/proj/src/main.c' \
  -c 'GccideFlags'
```

**Expect, in order:**
- A message line: `gccide: extracting compile flags from .../tests/fixtures/proj/Makefile ...`.
- Within ~1 s, a second message: `gccide: flags ready for .../tests/fixtures/proj (2 files)`.
- Vim does **not** block during extraction — try `:echo 'still alive'`
  immediately after `:GccideFlags`; it echoes without delay.

Leave this vim instance running for step 3.

## 3. Second `:GccideFlags` shows flags for the current buffer

In the same vim instance from step 2:

```
:GccideFlags
```

**Expect:** three lines printed:
- `root:  <REPO>/tests/fixtures/proj`
- `file:  <REPO>/tests/fixtures/proj/src/main.c`
- `flags: -Iinc -DPROJ_BUILD=1 -DVERSION="0.1" -std=c99 -Wall -Wextra`

(`<REPO>` is wherever you checked the project out.) Then `:qa!`.

## 4. On-disk cache file exists and is well-formed JSON

```sh
ls -l tests/fixtures/proj/.gccide/flags
cat tests/fixtures/proj/.gccide/flags | head -c 200; echo
```

**Expect:** the file exists; the JSON contains both `"mtime"` and
`"files"` top-level keys, and the file paths `src/main.c` and
`src/util.c` appear inside it. (Key order is alphabetical, so the
output starts with `{"files":...` — that is correct.)

## 5. Cache hit on a fresh vim invocation is instant (no `extracting`)

```sh
time vim80 -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit tests/fixtures/proj/src/main.c' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:**
- **No** `extracting ...` line in the output — flags load from
  `tests/fixtures/proj/.gccide/flags`.
- The `root:` / `file:` / `flags:` block prints immediately.
- Total wall time is well under 1 s (modulo vim's own startup).

## 6. Touching the Makefile invalidates the cache

```sh
touch tests/fixtures/proj/Makefile
vim80 -N \
  -c 'set rtp+=.' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit tests/fixtures/proj/src/main.c' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:** the `extracting ...` line returns, followed by
`gccide: flags ready ...`. The cache rebuilt because the mtime changed.

## 7. No Makefile in ancestry prompts the user

Pick a directory outside this repo that has no Makefile in its
ancestry — `/tmp` works almost everywhere. Then:

```sh
NOPROJ=$(mktemp -d)
echo 'int main(void){return 0;}' > "$NOPROJ/lone.c"
REPO=$(pwd)
vim80 -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "edit $NOPROJ/lone.c" \
  -c 'GccideFlags'
```

**Expect:** vim prompts `gccide: Makefile directory:` at the bottom of
the screen, with the temp directory as the default. Press Enter to
accept — the warning `gccide: no Makefile at <NOPROJ>` appears and the
command exits cleanly. `:qa!` to leave.

```sh
rm -rf "$NOPROJ"
```

## 8. `g:gccide_project_root` override

```sh
REPO=$(pwd)
vim80 -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c 'edit /tmp/anything.c' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:** `root:` line is the fixture path regardless of where the
buffer file lives. `flags:` falls back to the "no flags for this buffer"
message because `/tmp/anything.c` is not in the project — that is fine;
we only care that the root override took effect.

## Tear-down

```sh
rm -rf tests/fixtures/proj/.gccide
```

(The cache directory is git-ignored, but cleaning up between runs keeps
the next test honest.)

## If any step fails

Capture the exact output (including any `E…` errors) and append it to
`docs/JOURNAL.md` under a new dated entry before handing off.
