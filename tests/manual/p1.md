# P1 — Project detection + flag extraction: manual verification

Run these on the work laptop against a real Makefile-driven C/C++ project
(the firmware tree is the target). Substitute:

- `<REPO>`    — absolute path to this plugin's checkout.
- `<PROJ>`    — absolute path to a project root that contains a `Makefile`.
- `<SRC>`     — absolute path to a `.c` / `.cpp` / `.h` file *inside* `<PROJ>`.

The extraction is **async** (`job_start` + `make -Bnk`). On a large
firmware tree it can take several seconds. Wait for the
`gccide: flags ready ...` message before re-running `:GccideFlags`.

## 1. Plugin still loads and `:GccideStatus` still works

```
vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'GccideStatus' \
  -c 'qa!'
```

**Expect:** `alive` is printed. No `E*` errors.

## 2. `:GccideFlags` discovers the Makefile and extracts flags

```
vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit <SRC>' \
  -c 'GccideFlags'
```

**Expect:**
- First message line is `gccide: extracting compile flags from <PROJ>/Makefile ...`.
- After a few seconds, a second message appears:
  `gccide: flags ready for <PROJ> (N files)` with `N >= 1`.
- Vim is **not blocked** while extraction runs. Type `:echo 'still alive'`
  in the meantime and confirm it echoes immediately.

Leave this vim instance running for step 3.

## 3. Second `:GccideFlags` shows flags for the current buffer

In the same vim instance from step 2:

```
:GccideFlags
```

**Expect:** three lines printed:
- `root:  <PROJ>`
- `file:  <SRC>`
- `flags: ...` containing at least one `-I` or `-D` from your Makefile.
  (If `<SRC>` is a header not compiled directly, `flags:` may say
  `(none for this buffer; project has N files indexed)` — that is OK.)

Then `:qa!`.

## 4. On-disk cache file exists

```
ls -l <PROJ>/.gccide/flags
cat <PROJ>/.gccide/flags | head -c 200
```

**Expect:** the file exists and starts with `{"mtime":` followed by a JSON
object.

## 5. Cache hit on a fresh vim instance is instant

```
time vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit <SRC>' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:**
- No `extracting ...` line — flags load from `<PROJ>/.gccide/flags`.
- The `root:` / `file:` / `flags:` block prints immediately.
- Total wall time is well under 1 s (modulo vim's own startup).

## 6. Touching the Makefile invalidates the cache

```
touch <PROJ>/Makefile
vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit <SRC>' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:** `extracting ...` message appears again, followed by
`gccide: flags ready ...`. The cache rebuilt because the mtime changed.

## 7. No Makefile in ancestry prompts the user

Pick a directory outside `<PROJ>` that has no Makefile above it
(`/tmp` works on most systems). Create a throwaway `.c` file there:

```
echo 'int main(void){return 0;}' > /tmp/gccide_p1.c
vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'edit /tmp/gccide_p1.c' \
  -c 'GccideFlags'
```

**Expect:** vim prompts `gccide: Makefile directory:` with `/tmp` as the
default. Press Enter to accept (or type a path). If the path has no
`Makefile`, the warning `gccide: no Makefile at /tmp` appears and the
command exits cleanly. `:qa!` to leave.

```
rm -f /tmp/gccide_p1.c
```

## 8. `g:gccide_project_root` override

```
vim80 -u NONE -N \
  -c 'set rtp+=<REPO>' \
  -c 'runtime! plugin/gccide.vim' \
  -c 'let g:gccide_project_root = "<PROJ>"' \
  -c 'edit /tmp/anything.c' \
  -c 'GccideFlags' \
  -c 'qa!'
```

**Expect:** `root:` line in the output is `<PROJ>`, regardless of where
the buffer file lives. (`flags:` will fall back to the "no flags for this
buffer" message since `/tmp/anything.c` is not in the project — that is
fine; we only care that the root override took effect.)

## If any step fails

Capture the exact output (including any `E…` errors) and append it to
`docs/JOURNAL.md` under a new dated entry before handing off. Note
which step failed and what vim/`make` actually printed.
