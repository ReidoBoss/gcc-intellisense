# P6 — Performance: manual verification

P6 ships three changes on top of P3–P5:

1. **Incremental re-index on save.** `BufWritePost` on
   `*.c,*.cpp,*.cc,*.cxx,*.h,*.hpp,*.hh,*.hxx` calls
   `gccide#index#refresh_file(<afile>:p)`, which drops the file's
   entries from `s:idx`, re-extracts, splices the fresh symbols
   back, and re-persists. No full re-walk.
2. **mtime-gated `:GccideIndex`.** Before kicking off the full
   `find` walk we run `find . -newer <index> ... | head -n 1`
   async. Empty result → echo `gccide: index up to date` and bail.
   Otherwise → full rebuild via the existing path.
3. **Baseline timings.** Step 5 records `reltime()` numbers for
   the full build, single-file refresh, prefix-match completion,
   and exact-name lookup against the bundled fixture. They are
   tiny (microseconds — the fixture has 3 source files); the value
   is establishing the methodology so you can re-run on your real
   firmware tree and compare.

Prerequisites:
- `vim80` (vim 8.0) on PATH.
- `find` and `sh` — standard.

Steps 1–6 are scripted; step 7 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
rm -rf tests/fixtures/proj/.gccide
git checkout -- tests/fixtures/proj/src/util.c
```

## 1. Surfaces reachable

```sh
REPO=$PWD
vim80 -u NONE -i NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'runtime! autoload/gccide/index.vim' \
  -c 'echo exists("*gccide#index#refresh_file")' \
  -c 'echo exists("*gccide#index#build")' \
  -c 'qall' 2>&1
```

**Expect:** two `1` lines.

## 2. Full build prints stats, persists `.gccide/index`

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
cat > "$TMPDIR/gccide_p6_full.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideIndex
call gccide#index#_wait_done(10000)
call writefile(['stats=' . string(gccide#index#_stats())], '$TMPDIR/gccide_p6_full.out')
qall
VIM
rm -f "$TMPDIR/gccide_p6_full.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p6_full.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p6_full.out"
ls -la "$REPO/tests/fixtures/proj/.gccide/index"
```

**Expect:**
- `stats={'loaded_from': '…/.gccide/index', 'symbols': 5}`.
- The index file exists and is non-empty.

## 3. Second `:GccideIndex` short-circuits via mtime gate

```sh
REPO=$PWD
cat > "$TMPDIR/gccide_p6_gate.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideIndex
call gccide#index#_wait_done(10000)
qall
VIM
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p6_gate.vim" </dev/null 2>&1 | grep -E 'gccide:' || echo '(no echo)'
```

**Expect:** a `gccide: index up to date` echo. **No** `indexing …`
or `index built …` line — the find-newer check returned empty, so
we skipped the full build.

## 4. Incremental refresh adds and removes symbols

```sh
REPO=$PWD
cat > "$TMPDIR/gccide_p6_refresh.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'

" Load the persisted index (built in step 2).
let g:out = []
call add(g:out, 'before_proj=' . len(gccide#index#candidates('proj_')))

" Append a new function to util.c and refresh just that file.
let g:saved = readfile('$REPO/tests/fixtures/proj/src/util.c')
let g:new = g:saved + ['', 'int proj_brand_new(int x) {', '    return x * 2;', '}']
call writefile(g:new, '$REPO/tests/fixtures/proj/src/util.c')
call gccide#index#refresh_file('$REPO/tests/fixtures/proj/src/util.c')
call add(g:out, 'after_proj=' . len(gccide#index#candidates('proj_')))
call add(g:out, 'new_hit_count=' . len(gccide#index#lookup('proj_brand_new')))

" Restore the file and refresh again.
call writefile(g:saved, '$REPO/tests/fixtures/proj/src/util.c')
call gccide#index#refresh_file('$REPO/tests/fixtures/proj/src/util.c')
call add(g:out, 'restored_proj=' . len(gccide#index#candidates('proj_')))
call add(g:out, 'gone_hit_count=' . len(gccide#index#lookup('proj_brand_new')))

call writefile(g:out, '$TMPDIR/gccide_p6_refresh.out')
qall
VIM
rm -f "$TMPDIR/gccide_p6_refresh.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p6_refresh.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p6_refresh.out"
```

**Expect:**
```
before_proj=2
after_proj=3
new_hit_count=1
restored_proj=2
gone_hit_count=0
```

The `proj_` prefix originally matches two symbols (`proj_add`,
`proj_greet`). After appending `proj_brand_new` and refreshing
just `util.c`, the candidate count rises to 3 and the exact lookup
finds the new symbol. Restoring the file and refreshing again
drops the symbol cleanly.

## 5. Baseline timings (fixture-sized — record for posterity)

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
cat > "$TMPDIR/gccide_p6_time.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'

let g:out = []

" Full build (cold).
let t = reltime()
GccideIndex
call gccide#index#_wait_done(10000)
call add(g:out, 'full_build=' . reltimestr(reltime(t)) . 's')

" Single-file refresh.
let t = reltime()
call gccide#index#refresh_file('$REPO/tests/fixtures/proj/src/util.c')
call add(g:out, 'refresh_file=' . reltimestr(reltime(t)) . 's')

" 1000 omnicomplete candidate lookups on the 'proj_' prefix.
let t = reltime()
for i in range(1000)
  call gccide#complete#omnifunc(0, 'proj_')
endfor
call add(g:out, 'candidates_x1000=' . reltimestr(reltime(t)) . 's')

" 1000 exact-name lookups.
let t = reltime()
for i in range(1000)
  call gccide#index#lookup('proj_add')
endfor
call add(g:out, 'lookup_x1000=' . reltimestr(reltime(t)) . 's')

call writefile(g:out, '$TMPDIR/gccide_p6_time.out')
qall
VIM
rm -f "$TMPDIR/gccide_p6_time.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p6_time.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p6_time.out"
```

**Expect:** four lines like
```
full_build=  0.250000s
refresh_file=  0.000200s
candidates_x1000=  0.020000s
lookup_x1000=  0.010000s
```
(Exact numbers vary; the fixture is microscopic. The point is to
re-run this on your firmware tree and compare. The dominant cost
in `full_build` is the async `find` + `_wait_done` polling, not
parsing.)

## 6. BufWritePost integration: writing a buffer triggers refresh

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
git checkout -- "$REPO/tests/fixtures/proj/src/util.c"
cat > "$TMPDIR/gccide_p6_save.vim" <<VIM
set rtp+=$REPO
set noswapfile
filetype plugin on
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'

GccideIndex
call gccide#index#_wait_done(10000)

edit $REPO/tests/fixtures/proj/src/util.c
" Append a function to the live buffer, then :write — the
" BufWritePost autocmd should run refresh_file.
call append(line('\$'), ['', 'int proj_save_test(int n) {', '    return n + 1;', '}'])
write

let g:out = [
\ 'after_save_proj=' . len(gccide#complete#omnifunc(0, 'proj_')),
\ 'new_lookup=' . len(gccide#index#lookup('proj_save_test')),
\ ]
call writefile(g:out, '$TMPDIR/gccide_p6_save.out')
qall!
VIM
rm -f "$TMPDIR/gccide_p6_save.out"
vim80 -u NONE -i NONE -N -S "$TMPDIR/gccide_p6_save.vim" </dev/null >/dev/null 2>&1
cat "$TMPDIR/gccide_p6_save.out"
git checkout -- "$REPO/tests/fixtures/proj/src/util.c"
```

**Expect:**
```
after_save_proj=3
new_lookup=1
```

The `:write` inside vim fired the `BufWritePost` autocmd, which
called `gccide#index#refresh_file`, which spliced
`proj_save_test` into `s:idx`. The `git checkout` at the end
restores the fixture.

## 7. Interactive: type, save, complete a new identifier

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
git checkout -- "$REPO/tests/fixtures/proj/src/util.c"
vim80 -u NONE -i NONE -N \
  -c 'filetype plugin on' \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "let g:gccide_source_root  = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/util.c"
```

Then in vim:

1. `:GccideIndex` — wait for `gccide: index built (5 symbols, 3 files)`.
2. Go to the bottom of `util.c` (`G` in normal mode), open a new
   line (`o`), type:
   ```
   int proj_freshly_added(int x) { return x; }
   ```
   Press `Esc`.
3. `:w` — vim writes the file and you should see a brief delay
   (`refresh_file` running). No `index built …` echo this time;
   the refresh is silent on success.
4. On a new line in insert mode (`o`), type `proj_` then
   `<C-x><C-o>`. The popup includes `proj_freshly_added` alongside
   `proj_add` and `proj_greet`.
5. `:GccideFind proj_freshly_added` — the quickfix opens with the
   hit on the line you just added.
6. `:q!` to discard the edit, then from the shell:
   `git checkout -- tests/fixtures/proj/src/util.c` to restore
   the fixture.

## Known limitations (intentional for P6)

- **Refresh is silent on success.** No echo when `BufWritePost`
  triggers `refresh_file`, by design — you save dozens of times
  per session; the message would be noise. Failure modes (no
  in-memory + no on-disk index, file outside source root) are
  also silent — set `verbose` to debug if needed.
- **mtime gate is per-file, not per-content.** Saving a file with
  no semantic changes (just whitespace) still triggers the full
  rebuild path on the next `:GccideIndex`. Acceptable since the
  refresh-on-save handler already kept the index current; the
  full rebuild is a manual-only operation.
- **Deletions of source files are not detected** by the mtime gate.
  If you `rm foo.c` outside vim, `:GccideIndex` won't notice
  anything is newer than the index and will short-circuit, leaving
  `foo.c`'s entries in place. Workaround: `rm -rf .gccide/index`
  then `:GccideIndex` to force a clean rebuild. This is rare
  enough to not justify the extra `find` round.
- **`refresh_file` does not preempt itself.** Saving the same
  buffer twice in quick succession runs the extraction twice. The
  cost is small (single file, regex-only) — debouncing would add
  complexity for no measurable win at this size.

## If any step fails

Capture the exact output and append a dated entry to
`docs/JOURNAL.md`. Common gotchas:
- `g:gccide_project_root` / `g:gccide_source_root` unset.
- Tests run without `filetype plugin on` — the `BufWritePost`
  autocmd for `*.c,…` still fires (it's globbed on file name, not
  filetype), but the omnifunc autocmd from P4 needs `FileType c`.
- Fixture left in a modified state from step 6 or 7 — re-run
  `git checkout -- tests/fixtures/proj/src/util.c`.
