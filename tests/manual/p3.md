# P3 — Identifier index: manual verification

The plugin walks `g:gccide_source_root` (or, if unset,
`g:gccide_project_root`) for `.c`/`.cpp`/`.cc`/`.cxx`/`.h`/`.hpp`/`.hh`/`.hxx`
files, regex-extracts symbols (function definitions, `#define`s,
typedefs, struct/union/enum tags, file-scope globals) and persists the
result to `.gccide/index` (JSON) under the project root.
`:GccideFind <symbol>` populates the quickfix list with every hit.

Prerequisites:
- `vim80` (vim 8.0) on PATH.
- `find` and `sh` — standard.

Steps 1–6 are scripted; step 7 is interactive.

## Setup

```sh
cd /path/to/your/gcc-only-ide              # this repo's checkout
rm -rf tests/fixtures/proj/.gccide
git checkout -- tests/fixtures/proj/src/main.c
```

The bundled fixture exercises `#define` and function-definition
extraction (the kinds the project's real code uses most). The
extractor also handles typedef, struct/union/enum tag, and file-scope
global — those branches are covered by the per-file regex set but the
fixture intentionally keeps `inc/proj.h` minimal so P2's clean-build
sanity stays trivial.

`gccide#index#_wait_done(timeout_ms)` and `gccide#index#_stats()` are
test-only seams. In scripted (`vim80 -S`) mode plain `:sleep` does not
reap exited `find` children — `_wait_done` polls `job_status()` until
the build finishes.

## 1. Plugin still loads, :GccideStatus + :GccideIndex available

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'GccideStatus' \
  -c 'command GccideIndex' \
  -c 'command GccideFind' \
  -c 'qa!' 2>&1
```

**Expect:** `alive`, then two `Name … Args … Definition …` rows for
`GccideIndex` (0 args) and `GccideFind` (1 arg).

## 2. :GccideIndex builds, persists `.gccide/index`, prints stats

```sh
REPO=$PWD
rm -rf "$REPO/tests/fixtures/proj/.gccide"
cat > "/tmp/gccide_p3_build.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideIndex
call gccide#index#_wait_done(10000)
call writefile(['stats: ' . string(gccide#index#_stats())], '/tmp/gccide_p3_build.out')
qa!
VIM
rm -f "/tmp/gccide_p3_build.out"
vim80 -u NONE -N -S "/tmp/gccide_p3_build.vim" </dev/null >/dev/null 2>&1
cat "/tmp/gccide_p3_build.out"
echo --- index file ---
ls -la "$REPO/tests/fixtures/proj/.gccide/index"
```

**Expect:**
- `stats: {'loaded_from': '…/tests/fixtures/proj/.gccide/index', 'symbols': 5}`.
- `.gccide/index` exists and is non-empty.

The 5 symbols are: `PROJ_H`, `PROJ_MAGIC` (defines from `inc/proj.h`),
`main` (function in `src/main.c`), `proj_add` and `proj_greet`
(functions in `src/util.c`). Forward declarations in `inc/proj.h` are
correctly *skipped* — only definitions count as functions.

## 3. :GccideFind on a function definition

```sh
REPO=$PWD
cat > "/tmp/gccide_p3_find.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
GccideFind proj_add
call writefile(['qf: ' . string(getqflist())], '/tmp/gccide_p3_find.out')
qa!
VIM
rm -f "/tmp/gccide_p3_find.out"
vim80 -u NONE -N -S "/tmp/gccide_p3_find.vim" </dev/null >/dev/null 2>&1
cat "/tmp/gccide_p3_find.out"
```

**Expect:** one quickfix item with `'text': 'f: proj_add'`, `'lnum': 4`,
and `'filename'` (or resolved `bufnr`) pointing at
`tests/fixtures/proj/src/util.c`.

Note: this step does **not** call `:GccideIndex`. It exercises the
auto-load path that reads `.gccide/index` from disk on first
`:GccideFind`.

## 4. :GccideFind on each symbol kind present in the fixture

```sh
REPO=$PWD
cat > "/tmp/gccide_p3_kinds.vim" <<VIM
set rtp+=$REPO
set noswapfile
runtime! plugin/gccide.vim
let g:gccide_project_root = '$REPO/tests/fixtures/proj'
let g:gccide_source_root  = '$REPO/tests/fixtures/proj'
let g:out = []
for sym in ['PROJ_H', 'PROJ_MAGIC', 'main', 'proj_greet']
  call gccide#index#find(sym)
  let qf = getqflist()
  call add(g:out, sym . ': ' . (empty(qf) ? '(none)' : qf[0].text))
endfor
call writefile(g:out, '/tmp/gccide_p3_kinds.out')
qa!
VIM
rm -f "/tmp/gccide_p3_kinds.out"
vim80 -u NONE -N -S "/tmp/gccide_p3_kinds.vim" </dev/null >/dev/null 2>&1
cat "/tmp/gccide_p3_kinds.out"
```

**Expect:** exactly:
```
PROJ_H: d: PROJ_H
PROJ_MAGIC: d: PROJ_MAGIC
main: f: main
proj_greet: f: proj_greet
```

The leading letter is the `kind` field: `d` define, `f` function.
(The other extractor branches — `t` typedef, `s` struct/union/enum
tag, `g` global — exist in `autoload/gccide/index.vim` but are not
exercised by this minimal fixture; verify them against your own
codebase.)

**Expect:** `qf: []` — the empty quickfix list, plus a

## 6. :GccideIndex with neither root set warns and bails

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c 'GccideIndex' \
  -c 'qa!' 2>&1 | grep gccide
```

**Expect:** a line `gccide: g:gccide_source_root (or g:gccide_project_root) must be set`.

## 7. Interactive: :GccideFind opens quickfix and Enter jumps

```sh
REPO=$PWD
vim80 -u NONE -N \
  -c "set rtp+=$REPO" \
  -c 'runtime! plugin/gccide.vim' \
  -c "let g:gccide_project_root = '$REPO/tests/fixtures/proj'" \
  -c "let g:gccide_source_root  = '$REPO/tests/fixtures/proj'" \
  -c "edit $REPO/tests/fixtures/proj/src/main.c"
```

Then in vim:

1. `:GccideIndex` — wait for the `index built (5 symbols, 3 files)` message.
2. `:GccideFind proj_greet` — the quickfix window opens with one entry.
3. `Enter` on that entry — vim jumps to `src/util.c` at the line where
   `void proj_greet(const char *who) {` begins.
4. `:GccideFind PROJ_MAGIC` — quickfix shows the define at `inc/proj.h`.
5. `:qa!` to leave.

## Known limitations (intentional for P3)

- Multi-line function signatures (return type on its own line, args
  wrapped across lines) are recognised only when the closing `)`
  appears on the same line as the function name and the `{` is on the
  next non-blank line. Wider K&R-style wrapping is missed.
- Header guards (`#define FOO_H` after `#ifndef FOO_H`) are indexed
  like any other `#define`. Filtering them is a P6 concern at
  earliest.
- Globals are matched by a loose regex (file-scope identifier followed
  by `=` or `;` with no `(` on the line). A handful of edge-case
  declarations may be missed; that is acceptable for an
  identifier-only index.
- The index is fully rebuilt on every `:GccideIndex`. Incremental
  re-index on save is a P6 task.

## If any step fails

Capture the exact output (or the `:messages` buffer if interactive)
and append to `docs/JOURNAL.md` under a new dated entry before
handing off. Common gotchas:
- `g:gccide_project_root` (or `g:gccide_source_root`) unset.
- The Makefile-derived source list is **not** consulted by P3 — we walk
  the source root directly with `find`. If your `.c` files live
  outside the source root they won't be indexed; set
  `g:gccide_source_root` accordingly.
- Symbols inside `/* ... */` block comments that span more than one
  line may slip through if the depth tracker miscounts braces around
  the comment. Re-running `:GccideIndex` after closing such comments
  cleanly fixes it.
